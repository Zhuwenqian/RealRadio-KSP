# 功能更新

## 2026-07-04
- 实现神舟飞船真实无线电播报系统。
- 将 `Source/RealRadioMod.cs` 改为飞行场景入口 `[KSPAddon(Startup.Flight, false)]`，每次飞行独立初始化。
- 新增 `Source/ShenzhouRadioController.cs`：
  - 进入飞行场景后一次性检测神舟/CZ-2F 关键部件（服务舱、轨道舱、返回舱、一二级燃料箱）。
  - 判定为神舟飞船后启动无线电播报系统。
  - 当飞行器在轨（ORBITING）且所有引擎油门 ≤5% 时，判定无线电序列结束，后续不再触发音频。
- 新增 `Source/VesselEventTracker.cs`：
  - 追踪逃逸塔（`KCLV_CZ2F_Escape_tower`、`KCLV_CZ2F_Escape_tower_Upper`）。
  - 追踪一二级级间段（`KCLV_CZ2F_interstage`）。
  - 追踪整流罩（`KCLV_CZ2F_Fairing_Upper`、`KCLV_CZ2F_Fairing_Mid`）。
  - 当某类关键部件全部脱离当前飞行器时触发对应事件。
- 新增 `Source/RadioAudioPlayer.cs`：
  - 通过 `GameDatabase` 加载 `GameData/RealRadio/Radio Pack/Shenzhou/` 下的 `.ogg` 音频。
  - 支持单次播放与顺序播放（用于一二级分离的两段连播）。
- 映射音频事件：
  - 逃逸塔分离 → `escapetower deculop.ogg`
  - 助推器分离 → `booster deculop.ogg`
  - 一二级分离 → 依次播放 `stage1 deculoped1.ogg`、`stage1 deculoped2.ogg`
  - 整流罩分离 → `fairing deculop.ogg`
- 更新 `RealRadio.csproj`，加入新增的源文件，并添加 `UnityEngine.AudioModule` 引用以支持 `AudioSource` 与 `AudioClip`。
- 修复部件检测命名不匹配问题：所有查找用的神舟/CZ-2F 部件名统一改为点号格式（如 `KCHS.SZ.Re.entryModule`、`KCLV.CZ2F.stage1`），并在检测与事件追踪中对实际部件名做归一化（`Replace('_', '.')`），同时兼容 cfg 中的下划线写法。输出每个部件的原始名与归一化名以便调试。
